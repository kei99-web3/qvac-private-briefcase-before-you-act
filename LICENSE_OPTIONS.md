# License Options

Status: selected

## Recommended

Apache-2.0.

Reason: QVAC SDK is documented as Apache-2.0, so using Apache-2.0 for this project keeps the open-source posture aligned with the upstream SDK ecosystem.

## Acceptable Alternative

MIT.

Reason: The hackathon appears to accept permissive open-source licenses, and MIT is short and familiar.

## Decision

Apache-2.0 was selected for the public repository candidate. The actual `LICENSE` file is included at the repository root.
